1 · Mental model

Four layers, decoupled — you can swap any one without touching the others.

          ┌──────────────────────────────────────────────────────────┐
          │  STREAMLIT  ← interactive UI: pick → run → compare        │
          └────────────┬─────────────────────────────────────────────┘
                       │  data, signals, metrics
          ┌────────────▼─────────────────────────────────────────────┐
          │  BACKTEST   ← vectorbt: signals→portfolio, sweeps, stats  │
          └────────────┬─────────────────────────────────────────────┘
                       │  wide pandas frame (close[ticker], ...)
          ┌────────────▼─────────────────────────────────────────────┐
          │  STRATEGY   ← one file per signal: generate_signals()    │
          └────────────┬─────────────────────────────────────────────┘
                       │  polars DataFrame (long format)
          ┌────────────▼─────────────────────────────────────────────┐
          │  DATA       ← ClickHouse `bars` table  ←  YFinanceSource │
          └──────────────────────────────────────────────────────────┘
                       ▲                          ▲
                       │ direct CLI               │ Dagster asset (same code)
                       └──────────────────────────┘

The arrows tell you what depends on what. Strategies do not know about ClickHouse. The backtest engine does not know about yfinance. The dashboard knows nothing about Dagster. That decoupling is the whole point — it's what lets you iterate fast.

2 · The daily loop

What a typical research session looks like:

1
Start ClickHouse (once, lives across sessions): docker compose up -d
2
Refresh data if you haven't today: python scripts/backfill.py  or click Materialize in Dagster.
3
Open the dashboard: streamlit run dashboard/app.py
4
Sweep + compare. Pick a strategy, hit Sweep mode, eyeball the heatmap, note the top-3 by Sharpe.
5
Form a hypothesis ("RSI exit at 60 looks better than 50 on tech names — why?"). Write a new strategy file or tweak parameters.
6
Stop ClickHouse when done (optional): docker compose down. Data persists in a named volume.

How idempotency works

The bars table is a ReplacingMergeTree ordered on (ticker, interval, timestamp). After every insert we run OPTIMIZE TABLE bars FINAL DEDUPLICATE, and reads use FROM bars FINAL. Re-running a backfill never duplicates rows. Safe to schedule, safe to retry.

Inspect the database directly

# row counts per ticker:
curl 'http://localhost:8123/?query=SELECT+ticker,count()+FROM+algo.bars+GROUP+BY+ticker+ORDER+BY+ticker+FORMAT+PrettyCompact'

# or use the ClickHouse client inside the container:
docker exec -it algo-clickhouse clickhouse-client --query \
  "SELECT ticker, min(timestamp), max(timestamp), count() FROM algo.bars GROUP BY ticker ORDER BY ticker"

5 · Using the dashboard

Open localhost:8501. Sidebar on the left, results on the right.

What you'll see after pressing Run

  1. Metrics table — one row per backtest run, sorted by Sharpe descending. This is your shortlist.
  2. Equity curves overlay — every run's portfolio value on one chart. Visual sanity check.
  3. Price + entry/exit markers — the best run, plotted on the price of one selected ticker. Use this to spot regimes where the signal worked or whiffed.
  4. Sharpe heatmap — only appears in Sweep mode with exactly 2 params having more than one value. Read it like a contour map: green = high Sharpe, red = low.
Tip. When sweeping 3+ params, fix two of them in the param multi-selects (just pick one value) and let the other two vary. The heatmap then reveals the interaction between exactly those two.

6 · Driving it from code

The dashboard is one consumer of the backtest API. You can do the same thing in a notebook or script.

from data import load_bars
from backtest import polars_to_wide, run_backtest, sweep, compare
from strategies import MACrossover, RSIMeanReversion

# 1. Pull data
df = load_bars(['AAPL','MSFT','NVDA','SPY','QQQ'],
               start='2022-01-01', end='2024-12-31')
wide = polars_to_wide(df)

# 2. Single backtest
r1 = run_backtest(wide, MACrossover(fast=20, slow=50))
r2 = run_backtest(wide, RSIMeanReversion(period=14, oversold=30, exit_level=55))
print(compare([r1, r2]))

# 3. Sweep
results = sweep(wide, MACrossover,
                grid={'fast': [10, 20, 30], 'slow': [50, 100, 150]})
print(compare(results))

7 · Adding a strategy

One file. No engine changes. That's the contract.

1
Create strategies/momentum_breakout.py: 
import pandas as pd
from .base import Signals, Strategy

class MomentumBreakout(Strategy):
    name = "momentum_breakout"

    @classmethod
    def default_params(cls):
        return {"lookback": 20, "exit_lookback": 10}

    @classmethod
    def param_grid(cls):
        return {
            "lookback":      [10, 20, 50, 100],
            "exit_lookback": [5, 10, 20],
        }

    def generate_signals(self, data: pd.DataFrame) -> Signals:
        close = data["close"]
        high_n  = close.rolling(self.params["lookback"]).max()
        low_n   = close.rolling(self.params["exit_lookback"]).min()
        entries = (close >= high_n) & (close.shift(1) < high_n.shift(1))
        exits   = (close <= low_n)  & (close.shift(1) > low_n.shift(1))
        return Signals(entries.fillna(False), exits.fillna(False))
2
Register it in strategies/__init__.py: 
from .momentum_breakout import MomentumBreakout
REGISTRY["momentum_breakout"] = MomentumBreakout
3
Refresh the dashboard tab. The new strategy appears in the dropdown with its parameters already wired into Single & Sweep modes.

8 · Dagster orchestration

Dagster is optional. The bars asset is a thin wrapper that calls backfill_ticker() — the same function the CLI uses.

dagster dev -w workspace.yaml
# UI: http://localhost:3000

In the UI:

  • Assets → bars shows one partition per ticker. Click Materialize on any subset to backfill those tickers.
  • Schedules → daily_update_schedule is created in a STOPPED state by design. Toggle it on when you want the 21:30 UTC weekday refresh.
  • Jobs → daily_update_job can be triggered manually any time.
Why both CLI and Dagster? CLI for fast local iteration ("backfill this one ticker now"). Dagster for everything that needs lineage, retries, history, or a schedule. Same code under the hood — choose by ergonomics.

9 · Interpreting results

MetricWhat it tells youWatch for
SharpeRisk-adjusted return. >1 is good, >2 suspicious on short windows.Tiny trade counts inflate Sharpe artificially.
Total returnCumulative P&L over the period.Compare to buy-and-hold of the same tickers.
Max drawdownWorst peak-to-trough loss.Strategies that look great can hide 40%+ DDs.
Win rate% of closed trades that profited.Trend strategies can have low win rates but big winners.
n_tradesHow often you traded.<10 trades = statistically noisy. Sweep wider.
ExposureFraction of bars with a non-zero position.Very low = signal rarely fires. Very high = no edge over buy-and-hold.

Common research traps

  • Curve-fitting. The best Sharpe in a sweep is often just luck. Hold out a date range; re-test there.
  • Look-ahead bias. The framework uses signals from bar t to enter on bar t's close, which is fine because rolling() only sees past values. But if you write future_close.shift(-1) in a signal, you'll cheat. Don't.
  • Survivorship. Your universe is today's tickers. Backtests on NVDA since 2010 don't include the failed AI startups of that era.
  • Costs. Default is 5 bps each side. For high-turnover signals, raise it and watch Sharpe collapse — that's the real story.

10 · Troubleshooting

Dashboard says "ClickHouse unavailable"

docker compose ps — if not healthy, run docker compose up -d. If it crashed, docker compose logs clickhouse.

Dashboard says "No bars in ClickHouse for that selection"

You haven't backfilled that ticker / date range / interval. Run python scripts/backfill.py --tickers TICKER --start YYYY-MM-DD.

yfinance returns empty / 429-style errors

Rate-limited. Wait a minute, then retry. Or backfill fewer tickers at once. The YFinanceSource uses threads=True — if rate limits get worse, change to False in data/yfinance_source.py.

Sweep with N params blows up combinatorially

Cartesian product. 4 params × 4 values each = 256 runs. Trim the param multi-selects in the sidebar, or pass a smaller grid= in code.

Dagster UI port 3000 already in use

dagster dev -p 3001

Want to wipe and start over 
docker compose down -v       # ⚠ deletes the ClickHouse volume
docker compose up -d
python scripts/backfill.py

That's the platform. The fastest way to internalize it is to add one new strategy of your own — pick something simple like a Bollinger reversion or a 52-week-high breakout — and watch it appear in the dashboard. Once you've done that round-trip, the rest of the system will feel obvious.
